## Overview

This repository serves as the central documentation for our lab's shared server. The purpose of this guide is to assist all lab members in effectively utilizing server resources, managing software environments, and maintaining a stable, secure computing environment.

> [!IMPORTANT]
> **Safety First:** Before using the server, all members **must** read the [Precautions](docs/precautions.md#precautions--security) section. This is crucial for maintaining data integrity and server security. Failure to follow these guidelines may result in data loss or disruption of services for other users.
### What This Guide Covers

This documentation provides comprehensive details on:

*   **Software Inventory:** A list of installed software, their specific versions, and their research applications.
*   **Environment Management:** How to locate, activate, and utilize existing Conda environments.
*   **Shared Resources:** Instructions on how to install and share new Conda environments or software packages with other lab members.

### Documentation Structure

To keep the server organized, please refer to the following documents for specific tasks:

*   [**Precautions & Security**](docs/precautions.md): Mandatory reading regarding data safety, server etiquette, and resource management.
*   [**Software & Versions**](docs/software_list.md): A complete catalog of installed tools, their versions, and their research applications.
*   [**Environment Usage Guide**](docs/usage.md): Step-by-step instructions on activating environments and managing shared software.
*   [**Pipelines**](docs/pipeline.md): Example of some pipelines.
